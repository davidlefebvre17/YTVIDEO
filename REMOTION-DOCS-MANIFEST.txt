KEY INSIGHTS FOR YOUR PROJECT
================================================================================

For 100-image trading videos specifically:

Memory Optimization:
- Use TransitionSeries with Sequence to lazy-load images
- Only 1-2 images in RAM at once (not all 100)
- Memory usage: ~100 MB for rendering, manageable on any machine

Rendering Performance:
- Recommended concurrency: 6 (less than your CPU cores)
- H.264 codec: fastest for quality/speed trade-off
- Total render time: ~5-10 minutes for 8-10 minute video
- CRF 18: high quality, still reasonable file size

Beat Structure:
- Quick update scenes: 30-60 frames (1-2 seconds)
- Deep dive scenes: 60-120 frames (2-4 seconds)
- Transitions: 15-20 frames (spring animation)
- Total for 100 beats: ~3000-4000 frames (100-130 seconds @ 30fps)

Ken Burns Effect:
- Use 1.05-1.1x zoom for subtle effect (avoids blur)
- Start with static Ken Burns, add spring later
- Pair with 1-2 second frame duration for smooth feel

Transitions:
- slide({ direction: 'from-right' }) most professional
- springTiming({ durationInFrames: 15, damping: 12 }) bouncy but not too much
- Avoid too many different transition types (consistency = professional)

Sentiment/Importance:
- Color code by market sentiment (green = bullish, red = bearish)
- Size or opacity indicates importance
- Use data overlays (badges, charts) strategically
